  1. 函数
    函数：（function）功能，实现编译结构化，主要目的实现代码重用，在一个代码会在多次使用时，
    可以独立写成一个模块，在以下需要使用时，直接调用即可，但是函数不能作为执行入后，只能被其他程
    序调用才能执行，一个函数可以被多个程序调用

实现函数;
在 bash 中使用函数功能，需要把本来可以独立执行的一段代码封装成独立的一个功能，并取个
名
称，而后在需要使用此函数功能时，直接调用该函数名称即可

定义方法：
方法一：
function FUMNAME {
command
}
注解：
function 是关键字，必须要指定，不能错误

方法二：
FUMNAME() {
command
}

注解：
FUMNAME，命名规则，不能与变量名同样，只能包含字母、数字、下划线，而且不能以数字
开头，小括号()紧跟在使用 FUMNAME 后面，不能有空格，大括号括起需要执行的命令

例 1：
#!/bin/bash
function MENU {
cat << EOF
d|D) show disk usages
m|M) show memory userges
s|S) show swap userges
q|Q) quit.
EOF
}
MENU
read -p "You choice: " CHOICE
until [ $CHOICE == 'q' -o $CHOICE == 'Q' ]
do
case $CHOICE in
d|D) df -lh ;;
m|M) free -m |grep "^Mem" ;;
s|S) free -m |grep "^Swap" ;;
*)
echo -e "\n[d|D][m|M][s|S]"
;;
esac
MENU
read -p "You choice: " CHOICE
done

自定义函数执行状态返回值：
return #
0-255
在函数执行过程中，凡是遇到 return 都停止执行

例：
#!/bin/bash
ADDUSER() {
USERNAME=cjk
if ! id -u $USERNAME &> /dev/null
then
useradd $USERNAME &> /dev/null
echo $USERNAME |passwd --stdin $USERNAME &> /dev/null
return 0
else
return 1
fi
}
ADDUSER
echo $?

传递参数给函数：
在函数中使用$1、$2 . . . 引用位置变量，在调用函数时，FUNNAME VALUE VALUE . . .
例：
#!/bin/bash
TOSUM() {
echo $[$1+$2]
}
TOSUM 5 8

如果在执行脚本时指定参数时：
在函数中使用$1、$2 . . . 引用位置变量，在调用函数时，FUNNAME $1、$2 . . .

例 1：
#!/bin/bash
TOSUM() {
echo $[$1+$2]
}
TOSUM $1 $2

例 2：
判定某个网段的主机乊间那些可以 ping 通
1、使用函数实现每一台主机的判定过程 IP 为 192.168.0.1 - 192.168.0.10
2、在主程序中来调用函数范围内的所有主机的在线情况
#!/bin/bash
PINGHOST() {
if ping -c 1 -W 1 192.168.0.$1 &> /dev/null
then
echo "192.168.0.$1 OK"
else
echo "192.168.0.$1 ONT OK"
fi
}

for i in seq 1 10
do
PINGHOST $i
done

或：

#!/bin/bash
PINGHOST() {
if ping -c 1 -W 1 $1.$2 &> /dev/null
then
echo "$1.$2 OK"
else
echo "$1.$2 ONT OK"
fi
}
for i in seq $2 $3
do
PINGHOST $1 $i
done
在执行脚本时，需要传递 3 个参数，网段 起始主机 结束主机，例：./ping.sh 192.168.0 1 10

for ((expr1 ; expr2 ; expr3))
do
循环体
done

注解：(( expr1 ; expr2 ; expr3 ))
expr1，第一个表达式：用于指定初始条件，指定控制变量的初始值
expr2，第二个表达式：判定什么时退出循环
expr3，第二个表达式：修改 expr1 的变量的值

1、判断一个指定的 bash 脚本是否有语法错误，如果有则提醒用户键入 Q 或 q，无视错误并退出，
任意任意键可以通过 vim 打开指定的脚本
2、如果用户通过 vim 编辑后保存退出时仍然有错误，则重复第一步，如果编辑后没有语法错误，则
正常退出
#!/bin/bash
while ! bash -n $1 ; do
read -p "Syntax error,[Q|q] to quit others for editing:" CHOICE
case $CHOICE in
q|Q)
echo "Something wrong quiting."
exit 5
;;
*)
vim + $1
;;
esac
done
echo "not error"
